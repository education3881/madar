Prototype notes — for the 2026-06-13 discussion · stripped before implementation

  1. Orange means now. The current issue's numeral is the only orange numeral on the page — the same survey-marker semantics the design system gives orange on diagrams. When the Editor closes Issue 02, the numeral flips to ink and the block's ground flips to its era color. The page itself shows time. Does this match how you imagined "current vs. previous"?
  2. The grounds are your observation, adopted. Issue 01's block stands on sand (#E8E2D0) because its nine stills do; Issue 02 stands on paper (#FAFAF7) because its stills do. The drift you noticed becomes the system. Phase 2 (a pre-approved "ground ladder" — stone, clay — for Issues 03+) is in the Designer's memo; her recommendation is to hold until Issue 04 exists. Deployment-only, or open the ladder?
  3. Each issue gets one motif, never reused — Issue 01: the horizon (field-survey rule); Issue 02: the thread (two nodes, one line — matching, bridging). The motif lives on the issue divider here and the issue's grid tiles on Instagram, so the social grid and the site age together.
  4. Thumbnails are the real stills, referenced from /web/public/stills/ — open this file from inside the repo folder to see them. In production they're build-resolved.
  5. Implementation is specced, not started: an issue field in the article schema (24 frontmatter backfills), an /issues/ + /ar/issues/ page pair iterating the collection (never naming slugs), and the homepage's hardcoded "Issue 01" label replaced by the derived value — that label is currently drift-prone hand-text. Sequencing is binding: this lands after the domain cutover, never straddling it. Naming check: "Issues" / «الأعداد» — or "Editions"?
