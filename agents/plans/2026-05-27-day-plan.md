## 1 · Editor — clear placeholder source URLs on Pieces B and C

**Why first:** The two "Draft — pending source verification" pills are the most visible quality artifact on the live site. Removing them is the single highest-leverage edit available tomorrow — it converts a v0.1 that *looks* draft into one that reads as finished.

**Scope (precise):**
- Piece B — `2026-05-25-jordan-double-shift` (curated, Levant)
- Piece C — `2026-05-25-colombia-escuela-nueva` (curated, LATAM)
- Piece A (Sierra Leone) is editorial, not curated, and is **not in scope** for this pass.

**What "verified" means:**
- Source is a primary document (ministry memo, peer-reviewed paper, named local journalism, named educator quoted in a named publication).
- URL resolves on first try.
- Publication date is correct to the day where the source carries one.
- The title rendered on the page matches the title on the source page.

**Workflow:** Editor opens each piece in `web/src/content/articles/`, replaces each `https://example.com/placeholder-*` URL and any `PLACEHOLDER SOURCE` titles with verified values, leaving the rest of the frontmatter alone. The article template detects the change and the pill drops automatically; no template work needed.

**Deliverable:** Two updated markdown files, committed with a single message `editor: verify sources for Jordan + Colombia curated pieces`.

**Gate:** Manager spot-checks one source per piece (URL resolves; title matches). No founder gate — this is a quality cleanup, not an editorial decision.

**Time box:** 90 minutes. If a source can't be verified inside that window, the piece reverts to *Draft — pending source verification* state and the failure is reported to Vini with the specific source flagged.
