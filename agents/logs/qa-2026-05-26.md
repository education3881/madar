## Defects filed and their disposition

### P0-001 — Wordmark broken on home page (EN + AR)

- **Reported:** founder this morning ("the logo is not working on the main page")
- **Root cause:** wordmark loaded via `<img src>` instead of inlined; `currentColor` does not inherit through `<img src>` so the theme color was lost; the asset depended on a separate network round-trip with no fallback if it failed.
- **Fix:** switched `web/src/pages/index.astro` masthead to inline the SVG at build time, using the same loader pattern as `SiteHeader.astro`. Added a typographic CSS fallback so even if the SVG were missing the brand still reads. Applied the same fix to `web/src/pages/ar/index.astro` with multi-candidate path resolution to defend against subdirectory path differences.
- **Status:** Fixed in source. Verified in local build. Awaiting deploy.
- **Lesson:** Brand-critical chrome is inlined, not loaded. Codified in the new Web Developer persona as the **"never ship without inlining" rule**.

### P0-002 — Live Arabic home links to parked Jordan piece

- **Reported:** found in this QA pass
- **Root cause:** the Arabic placeholder copy and the "Arabic excerpts from Jordan piece" block were both hardcoded on the assumption that the Jordan piece would always be available.
- **Fix:** removed both the Arabic paragraph that named the Jordan piece and the excerpts block. Replaced with a single Arabic line acknowledging the issue is in preparation.
- **Status:** Fixed in source. Verified in local build. Awaiting deploy.
- **Lesson:** Any page that hardcodes references to a specific piece needs to be regenerated when that piece is parked. Adding to the QA checklist: search the whole codebase for the parked piece's slug before considering a parking "done."

### P1-001 — `/madar/rss.xml` referenced in footer but not built

- **Reported:** found in this QA pass
- **Root cause:** the SiteFooter component links to an RSS feed that the Astro build does not currently produce. Likely intended for the Astro RSS plugin (`@astrojs/rss`), which is not installed.
- **Fix proposed:** install `@astrojs/rss`, add `src/pages/rss.xml.js` that reads the articles collection and emits a valid RSS feed. Estimated time: 30 minutes plus a build verification.
- **Status:** OPEN. Logged as a Web Developer backlog item, P1 (visible-but-broken footer link); fix scheduled for tomorrow's deploy window.
