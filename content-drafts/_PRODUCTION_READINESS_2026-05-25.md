## Schema mismatches that apply to all three drafts

The drafts were written against the editorial convention, not against the Astro schema at `/web/src/content.config.ts`. The Web Developer should adapt the frontmatter when moving the files. Specifically:

- `standfirst:` in the drafts maps to `dek:` in the schema. The text already complies with the schema's `max(200)` character limit on all three pieces (I checked).
- `themes:` in the drafts is an array of strings — already a match for the schema's `z.array(z.string()).min(1)`. No change needed.
- `sources:` in the drafts is currently a flat list of strings (e.g., `"Ministry of Basic and Senior Secondary Education, Annual School Census 2023…"`). The schema expects `z.array(z.object({ title, url }))` with a valid URL on each. The Editor will supply working URLs once Vini signs off on the source verification round; for the v0.1 publication wiring, the Web Developer should either (a) restructure the sources as `{ title, url }` objects with the existing strings as `title` and a verified URL paired in, or (b) coordinate with Editor to fill the URLs in the same pass that moves the files. Option (b) is cleaner.
- `region:` is required by the schema (`MENA | Africa | Asia | LatAm-Caribbean | Other`) but is missing from all three drafts. Add: Sierra Leone → `Africa`, Jordan → `MENA`, Colombia → `LatAm-Caribbean`.
- `level:` is required (`ECE | K-12 | Both`) and missing from all three. Add: Sierra Leone (Piece A, about Primary 3) → `K-12`; Jordan (Piece B, about K-12 double-shift schools) → `K-12`; Colombia (Piece C, about multi-grade primary) → `K-12`. None of the three v0.1 pieces is ECE — that gap is noted for the calendar.
- `type:` is required (`curated | short | essay`). The drafts use `editorial`, `curated`, `curated`. Map `editorial` → `short` for Piece A; the other two already match `curated`.
- `approved:` defaults to `false` in the schema, which is correct for v0.1 since Vini has not yet signed off.
- `hero_brief_for_designer:` is editorial metadata, not a schema field, and should be **stripped** from the file when moved. The hero data goes under the schema's `hero: { src, alt, caption }` object once the Designer's Still SVGs are inlined or referenced.
- `contains_composites:` and `composites:` are present on Piece A and are editorial tracking fields, not in the current schema. The Web Developer should either (i) extend the schema to include them as optional fields (preferred — they exist for a reason) or (ii) preserve them as comments above the frontmatter block so they aren't lost. The dagger disclosure in the body is what the reader sees; the frontmatter flag is for the Editor's audit trail.
